## 4. V2 学習フロー全体

### 4.1 V2 アーキテクチャ原理

```
VoiceConversionWrapper
├── content_extractor_narrow   (AstralQuantizer: Whisper-small + HuBERT → narrow content)
├── content_extractor_wide     (同上 → wide content)
├── style_encoder              (CAMPPlus: 参照音声 → 192次元 speaker embedding)
├── CFM  ← ★ 微調整可能（flow matching 拡散 denoiser, 337MB）
│   └── cfm_length_regulator
├── AR   ← ★ 微調整可能（自己回帰 Transformer-12層, 359MB）
│   └── ar_length_regulator
└── vocoder                    (BigVGAN 22kHz, 凍結)

推論: source音声 → content_extractor → CFM(style条件) → AR(フレーム単位生成) → BigVGAN → 出力
```

**CFM** は音色マッピング（「花丸の声がどういう音か」）を担当する。
**AR** は発音リズム（「花丸の話すリズム/間の取り方」）を担当する。

### 4.2 話声データ分割アルゴリズム

雑談/配信アーカイブ向けの逆向き目標点分割法：

```
1. silencedetect (-35dB, 0.3s) → ≥0.3s の本当の停止だけを取得
2. ファイル ≤30s → そのまま保持
3. ファイル >30s:
   ├─ IDEAL_SEG=15s に基づき均等な目標切断点を生成: 15s, 30s, 45s, ...
   ├─ 各目標点付近 [target-5, target+10] で停止を検索 → dur 最大を選択
   ├─ ウィンドウを [target-8, target+15] → [target-10, target+20] へ拡大
   ├─ 区間長 ≤30s → normal/ へ出力
   └─ 区間長 >30s → manual/ へ出力（_OVER_XXs suffix 付き）
```

**主要パラメータ：**
- ノイズ閾値：`-35dB`（UVR 後のドライ音声はクリーン）
- 最短無音：`0.3s`（文間の息継ぎ/思考停止）
- 理想区間長：`15s`
- 最大区間長：`30s`

**密な話声の配信アーカイブ向け：** `-20dB, 0.2s`

**結果：**
- 22 個の話声ファイル（8.1 時間）→ 正常 1695 区間（436 分）← V2 学習データ
- 巨大配信ファイル（135min/60min など）は `recut_dense.py` で個別処理

### 4.3 V2 学習成果

| 回 | データ | モジュール | ステップ数 | run name | VRAM | 速度 |
|------|------|------|------|----------|------|------|
| V2 R1 | 1695 件の話声 | CFM only | 6000 | hanamaru_v2_cfm | ~8GB | ~4.5 it/s |
| V2 R2 | 1695 件の話声 | CFM + AR | 6000 | hanamaru_v2_cfm_ar | **11.4GB** | ~2.5 it/s |

**CFM checkpoint:** 336MB/個、12 個で計 4GB
**AR checkpoint:** 318MB/個、12 個で計 3.8GB

### 4.4 V2 主要 Bug 修正

1. **CAMPPlus BatchNorm クラッシュ** — `vc_wrapper.py` の `forward()` 内で `style_encoder` を `eval()` モードにする必要がある（batch=1 学習時、BN は ≥2 サンプルを要求）
2. **CFM タイムスタンプ squeeze** — `cfm.py` の学習パスで `t.squeeze()` → `t.squeeze(1).squeeze(-1)`（shape 不一致）
3. **AstralQuantizer オフラインモード** — 元の `local_files_only=False` に戻し、ネットワーク接続時は自動ダウンロード
4. **HuBERT キャッシュ破損** — 削除して再ダウンロード（`preprocessor_config.json` 欠落）
5. **train_v2.py の checkpoint 自動削除** — `max_keep=1` を全保持へ変更

## 8. FAQ

**Q: 呼吸に電子音が乗る場合は？**
A: V1：`--diffusion-steps` を 4 から 20 に上げる。V2 はもともと電子音が出ない。

**Q: 音色があまり似ない場合は？**
A: V1：学習データ/ステップ数を増やす、または `--inference-cfg-rate` を 1.0 まで上げる。

**Q: 音程が外れる場合は？**
A: V1 では `--f0-condition True` を確認する。V2 では解決できないため V1 に切り替える。

**Q: V2 を歌唱に使える？**
A: 使えない。V2 には F0 誘導がないため、歌唱では必ず音程が外れる。

**Q: V1 は 100 歩のほうが 20 歩より良い？**
A: RTF は 0.7→2.0 になるが、聴感差はごく小さい。20 歩が限界効用上の最適点。

**Q: V2 similarity-cfg-rate はいくつにすべき？**
A: 微調整後は 0.1~0.3（モデル重み主導）、ゼロショットでは 0.7（参照音声主導）。

**Q: CFM と AR はそれぞれ何を担当する？**
A: CFM は音色（「誰の声に似るか」）、AR はリズム/発音パターン（「どう話すか」）を担当する。

**Q: VRAM 11.4GB で CFM+AR を回すのは安全？**
A: ぎりぎりで余裕はない。num-workers を有効にせず、他の GPU アプリを同時に動かさない。

**Q: RTF とは？**
A: Real-Time Factor。RTF=0.14 は 1 秒の音声を 0.14 秒で生成するという意味で、リアルタイムより 7 倍速い。
